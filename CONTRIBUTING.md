# Contributing to Ledge

Ledge is a small, solo-maintained project. Contributions are welcome — this guide keeps things lightweight.

## Reporting bugs

Open a [🐛 Bug Report](.github/ISSUE_TEMPLATE/bug_report.yml). Include clear steps to reproduce, expected behavior, and your environment (macOS + Ledge version).

## Requesting features

Open a [✨ Feature Request](.github/ISSUE_TEMPLATE/feature_request.yml). Describe the problem you're trying to solve and your proposed solution.

## Tracking work

Planned work, ideas, and backlog items live as [GitHub Issues](../../issues). Check there before starting something to avoid duplicate effort. Use the [📌 Task](.github/ISSUE_TEMPLATE/task.yml) form for internal or cleanup tasks.

## Labels

Issues are organized with a small set of labels:

- `bug` — something is broken
- `feature` — a new capability
- `task` — a tracked chore or backlog item
- `enhancement` — an improvement to existing behavior
- `documentation` — docs-only changes
- `good first issue` — approachable for newcomers
- `help wanted` — maintainer is looking for help

These are applied automatically by the issue forms where relevant; you can adjust them as needed.

## Contributing code

1. Fork and create a branch from `main`.
2. Make your change, following the repo's TypeScript strict style and Conventional Commits.
3. Run `pnpm lint` and `pnpm test`.
4. Open a pull request using the provided template.

No CLA, no GitHub Projects, no heavy process — just clear issues and clean PRs.
